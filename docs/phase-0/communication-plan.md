# Communication Plan

**Project:** CCSHAU Official Website  
**Last updated:** 23 June 2026

## 1. Communication Objectives

- Keep Computer Section and stakeholders informed of progress
- Surface blockers and dependencies early
- Document decisions and approvals in writing

## 2. Meetings

| Meeting | Frequency | Participants | Purpose |
|---------|-----------|--------------|---------|
| Kick-off | Once | All stakeholders | Charter, scope, roles |
| Weekly status | Weekly | PM, Computer Section, Dev lead | Progress, risks, decisions |
| Design review | Phase 1 | PM, UI/UX, Computer Section | 3 layout options → 1 approved |
| Sprint demo | Bi-weekly (from Phase 3) | Dev team + Computer Section | Working software demo |
| Migration review | Phase 5 | Dept content owners | Content validation |
| UAT session | Phase 7 | University reviewers | Test cases, defects |
| Go-Live readiness | Pre-launch | All key stakeholders | Cutover checklist |

## 3. Reporting

| Report | Frequency | Recipient | Template |
|--------|-----------|-----------|----------|
| Weekly progress report | Weekly | Incharge, Computer Section | [weekly-report-template.md](./weekly-report-template.md) |
| Risk register update | Weekly | PM, Computer Section | [risk-register.md](./risk-register.md) |
| Clarification log | As needed | PM, Sponsor | [clarification-tracker.md](./clarification-tracker.md) |

## 4. Channels

| Channel | Use |
|---------|-----|
| Email | Formal approvals, weekly reports |
| Shared document folder | Governance docs, inventory, UAT tracker |
| Issue tracker (GitHub Issues / Jira) | Defects, tasks, change requests |
| Video call | Workshops, demos, training |

## 5. Escalation Path

1. **Developer / Designer** → Development Lead  
2. **Development Lead** → Project Manager  
3. **Project Manager** → Incharge, Computer Section  
4. **Computer Section** → Director (Store & Purchase) — scope, timeline, contract

## 6. Response Times (Target)

| Priority | Description | Response | Resolution target |
|----------|-------------|----------|-------------------|
| P1 | Go-Live blocker, security issue | 4 hours | 1 business day |
| P2 | UAT defect, migration error | 1 business day | 3 business days |
| P3 | Minor UI, content formatting | 2 business days | Next sprint |
